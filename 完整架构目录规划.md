# 设计院数据资产保护平台 — 完整架构目录规划

## 五层总体架构

```
┌──────────────────────────────────────────────────────────────┐
│  Layer0: 运维监控层                                            │
│  Jenkins + Prometheus/Grafana + ELK                           │
├──────────────────────────────────────────────────────────────┤
│  Layer1: Web 应用层                                           │
│  Nginx + PHP(Laravel) + Samba                                 │
├──────────────────────────────────────────────────────────────┤
│  Layer2: 存储层                                               │
│  ZFS 存储池 + NFS 共享                                        │
├──────────────────────────────────────────────────────────────┤
│  Layer3: 数据库层                                             │
│  MySQL MGR 一主两从                                           │
├──────────────────────────────────────────────────────────────┤
│  Layer4: 代码与配置版本控制层                                    │
│  Git (GitHub/GitLab) + 制品管理                                │
└──────────────────────────────────────────────────────────────┘
```

## 四、部署顺序建议

| 阶段 | 时间 | 部署内容 | 目标 |
|------|------|---------|------|
| **Phase 1** | 第1-2周 | Layer1: Web 环境强化 | 完成当前 LNMP + Samba 基础 |
| **Phase 2** | 第3-4周 | Layer4: Git 仓库搭建 | 所有代码/配置/文档进 Git |
| **Phase 3** | 第5-6周 | Layer0: Jenkins | CI/CD 自动部署/备份 |
| **Phase 4** | 第7-8周 | Layer3: MySQL MGR | 数据库高可用集群 |
| **Phase 5** | 第9-10周 | Layer2: ZFS 存储 | 冷热分层 + 快照 |
| **Phase 6** | 第11-12周 | Layer0: Prometheus+Grafana | 全栈监控 + 告警 |
| **Phase 7** | 第13-14周 | Layer0: ELK | 日志集中 + 审计 |
| **Phase 8** | 第15-16周 | 集成联调 | 所有层打通测试 |
| **Phase 9** | 第17-20周 | 文档 + 答辩准备 | 架构图/演示/PPT |
